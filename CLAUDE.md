# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Development commands

```bash
# Start dev server (Python 3, no dependencies)
python serve.py          # → http://localhost:8080/web/
                         # → http://localhost:8080/web/test.html  (unit tests)

# Download ONNX Runtime WASM files (one-time setup)
bash web/setup.sh

# Convert nnU-Net model to ONNX (requires nnunetv2 torch onnx onnxruntime)
python scripts/convert_model.py \
    --model-dir /path/to/MELD-PostOp/model \
    --out web/models/meld_postop.onnx \
    --fold 0          # omit --fold for all 5 folds
```

**Running tests**: open `http://localhost:8080/web/test.html` in a browser while the dev server is running. All 7 suites run automatically on page load. There is no CLI test runner — the test page loads live browser JS including ONNX Runtime WASM.

**Regenerating test fixtures** (Python, requires onnxruntime + skimage):
```bash
python scripts/create_test_data.py           # preprocessing fixtures
python scripts/create_inference_test_data.py # ONNX inference fixtures (needs fold 0 model)
```

## Architecture

### All inference runs in a Web Worker (`web/worker.js`)

The main thread (`web/main.js`) owns the NiiVue viewer and UI; it sends `{type:'run', inputData}` to the worker and receives `{type:'stageData', data}` back with the encoded NIfTI mask. The worker is the only place that touches ONNX Runtime.

The worker pipeline maps directly to nnU-Net v2's `predict_single_npy_array()`:
1. `parseNifti` (nifti.js) → float32 data + header
2. `resampleTrilinear` (preprocess.js) → 1 mm isotropic target spacing
3. `foregroundZScore` (preprocess.js) → z-score normalisation on non-zero voxels
4. `slidingWindowInference` (worker.js) → softmax accumulation with Gaussian weighting
5. `logitsToSegmentation` (preprocess.js) → argmax + nearest-neighbour back-resample

### Axis convention throughout: (z, y, x) / (D, H, W)

`parseNifti` returns `dims=[nz,ny,nx]` and `pixdim=[sz,sy,sx]` — the z-axis (slice direction) is first, matching SimpleITK / nnU-Net conventions. NIfTI-1 stores x-fastest in memory, so `flat_index = z*ny*nx + y*nx + x` — **no data transposition is needed**, only label reordering.

### Coordinate convention: endpoint-aligned

Both `resampleTrilinear` and `resampleNearestNeighbor` use `src = dst * (sN-1)/(dN-1)`, which maps endpoint 0 → 0 and endpoint dN-1 → sN-1. This matches `skimage.transform.resize` (the function nnU-Net's `DefaultPreprocessor` uses). Using `sN/dN` (center-of-pixel) instead produces wrong results at boundaries.

### SharedArrayBuffer requirement

ONNX Runtime Web with WASM threading requires `SharedArrayBuffer`, which the browser only allows when the page is served with `Cross-Origin-Opener-Policy: same-origin` and `Cross-Origin-Embedder-Policy: require-corp`. `serve.py` injects these headers on every response. `web/coi-serviceworker.js` handles the case where the server cannot set them (e.g. GitHub Pages).

**Why not WebGPU?** The ONNX Runtime Web WebGPU EP explicitly lists "ConvTranspose3d is not supported" in its operator table (confirmed up to v1.26). The nnU-Net 3d_fullres decoder uses 3D `ConvTranspose` throughout, so WebGPU fails at inference time with "currently only support 2-dimensional conv". WASM is the only viable backend for this model until the WebGPU EP adds 3D conv support.

### NiiVue overlay display

Overlay volumes must be loaded via `nv.loadVolumes([base, mask])` — in v0.46.0, `nv.addVolume(plainObject)` expects an `NVImage` instance, not a descriptor, and fails silently. The mask header must explicitly set `cal_max=1` / `cal_min=0` (bytes 124/128) because `encodeNifti` copies the T1w header verbatim and the T1w's display window would map the {0,1} mask values to near-invisible intensities.

### Model files

`web/models/meld_postop_fold{0..4}.onnx` — not in the repository. Generated by `scripts/convert_model.py` from the MELD-PostOp Figshare download. The worker currently loads fold 0 only (`MODEL_URLS = [0].map(...)`); change to `[0,1,2,3,4]` for the full 5-fold ensemble.

### Test data

`data/tests/` holds NIfTI files and JSON fixtures generated by the `scripts/create_*.py` scripts. The browser test suite (`web/test.html`) fetches these at runtime and compares JS outputs to the Python ground truth. Tolerances reflect real float32 accumulation differences between CPU onnxruntime and WASM (boundary voxels of zero-padded patches can differ by up to ~0.15 logit units).
